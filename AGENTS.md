# RULES — обязательны для каждого ответа

## Скоуп
- Делай ТОЛЬКО то, что явно запрошено. Ничего «заодно».
- Запрещено: новые зависимости, новые файлы, переименования,
  рефакторинг соседнего кода — без явного разрешения.
- Если задача требует изменить >3 файлов или >80 строк — СТОП,
  покажи план и жди подтверждения.

## Код
- Минимальный диф. Редактируй, а не переписывай файлы.
- Переиспользуй существующие утилиты (src/lib, src/hooks) — сначала ищи, потом пиши.
- Никаких «креативных» названий (Engine, Fortress, Paranoid, Defiant).
  Имена — скучные и описательные.
- Ни одной фичи в UI без реализованной логики за ней.
- TypeScript strict, никаких `any` в новом коде.

## Процесс
- Перед кодом: 3–5 строк плана (какие файлы, что именно).
- После кода: команда проверки (tsc/build/test), которую я должен запустить.
- Если не уверен — спроси, а не придумывай.
